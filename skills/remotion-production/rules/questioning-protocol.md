# Questioning Protocol — 질문 프로토콜

이 패키지를 사용하는 에이전트는 영상 제작에 필요한 정보가 충분할 때까지 **계속해서 질문**해야 합니다.

## 핵심 원칙

### 1. 추측 금지 (Never Assume)
사용자가 명시하지 않은 정보를 스스로 결정하지 않습니다. 모르면 반드시 물어봅니다.

### 2. 모호함 감지 (Detect Ambiguity)
답변이 다음에 해당하면 **모호한 답변**으로 판단하고 구체화 질문을 합니다:

**모호 신호 키워드:**
- "적당히", "알아서", "그냥", "대충", "아무거나", "상관없어", "몰라", "괜찮은 걸로"
- "좋은 느낌으로", "예쁘게", "멋지게", "깔끔하게", "자연스럽게"
- 5단어 미만의 너무 짧은 답변
- "네" 또는 "아니오"만의 답변 (맥락상 더 구체적인 정보가 필요한 경우)

**구체화 질문 방법:**
> 사용자: "분위기는 적당히 좋게 해줘"
> 에이전트: "분위기를 좀 더 구체적으로 정해볼게요. 다음 중 어떤 느낌이 가장 가까운가요?
> 1. 밝고 에너지 넘치는 느낌 (광고, 홍보)
> 2. 차분하고 신뢰감 있는 느낌 (기업, 교육)
> 3. 긴장감 있고 시네마틱한 느낌 (다큐멘터리, 안전)
> 4. 따뜻하고 감성적인 느낌 (브랜딩, 스토리)"

**선택지 제시 원칙:** 모호한 답변에 대해서는 반드시 구체적인 선택지를 2~4개 제시하여 사용자가 고르기 쉽게 합니다.

### 3. 점진적 깊이 (Progressive Depth)
한꺼번에 20개를 질문하지 않습니다. 단계별로 점점 깊은 질문을 합니다:
- 1차: 큰 방향 (목적, 대상, 분위기)
- 2차: 구체적 선택 (스타일, 색감, 구조)  
- 3차: 세부 디테일 (각 장면의 인물, 움직임, 카메라)

### 4. "알아서 해줘" 대응
사용자가 "알아서 해줘"라고 할 경우:
1. 구체적인 제안을 **2~3가지** 제시
2. 각 제안의 차이점을 설명
3. 하나를 선택하게 함
4. 절대 무응답 상태로 진행하지 않음

### 5. 문서 작성 전 정보 완성도 검증
문서를 작성하기 전에 해당 문서에 필요한 모든 항목이 사용자로부터 확보되었는지 체크합니다:
- 비어있는 항목이 있으면 질문
- 이전 답변과 충돌하는 부분이 있으면 확인
- 새로운 결정이 필요한 부분이 있으면 질문

### 6. 맥락 기반 추가 질문
이전 단계의 답변을 바탕으로 후속 질문을 동적으로 생성합니다:
- 인물이 등장한다고 했으면 → 인물 상세 (복장, 표정, 연령대)
- 실외라고 했으면 → 시간대, 날씨, 계절
- 특정 업종이라고 했으면 → 해당 업종의 시각적 관습, 규제

## 질문 형식 가이드

### 좋은 질문 (구체적, 선택지 포함)
> "나레이션 화자의 톤은 어떤 느낌이면 좋겠나요?
> 1. 뉴스 앵커처럼 권위 있고 공식적인 톤
> 2. 선생님처럼 친근하고 설명하는 톤  
> 3. 동료처럼 편안하고 대화하는 톤"

### 나쁜 질문 (너무 열린 질문)
> "나레이션은 어떻게 할까요?"

### 좋은 후속 질문 (이전 맥락 연결)
> "아까 안전 교육 영상이라고 하셨는데, 사고 장면을 재현하는 부분이 필요한가요? 아니면 올바른 절차만 보여주는 방식이 좋을까요?"

## 각 커맨드별 적용

이 프로토콜은 모든 프리프로덕션 커맨드에 적용됩니다:
- `/receive-brief`: 10개 필수 항목 + 모호함 감지 + 후속 질문
- `/creative-brief`: 전략 결정에 대한 질문 (톤, 메시지, KPI)
- `/treatment`: 비주얼 선호도 질문 (스타일, 색감, 레퍼런스)
- `/write-script`: 나레이션 스타일, 장면 구성, 강조 포인트 질문
- `/storyboard`: 장면별 8개 항목 + 모호함 감지
- `/prompt-sheet`: 최종 확인 질문 (AI 프롬프트 방향)
