# Agent Orchestration — 서브에이전트 병렬 처리 규칙

## 아키텍처

```
사용자 → 메인 에이전트 (오케스트레이터)
            │
            ├→ media-scout (서브에이전트)
            │    이미지 생성 + 영상 클립 생성 + 스톡 검색 병렬 처리
            │
            ├→ post-producer (서브에이전트)
            │    AI 리뷰 + QC 검증 + 접근성 검사 병렬 처리
            │
            └→ 메인 에이전트 직접 수행
                 나레이션 생성, 음악 생성, 코드 작성, 렌더링
```

## 병렬 처리 구간

### `/create-video` 파이프라인

```
Phase 1: 문서 확인 + 설정 확인              [순차 — 메인]
    ↓
Phase 2: 오디오 생성
    2a. 나레이션 생성                        [순차 — 메인] ← 타이밍 기준
    2b. 배경음악 + 효과음                    [병렬 — 메인]
    ↓
Phase 3: 비주얼 에셋 수집                    [병렬 — media-scout 위임]
    ├→ 스톡 영상 검색 + 다운로드
    ├→ AI 이미지 생성
    └→ AI 영상 클립 생성
    ↓
Phase 4: Remotion 컴포지션 작성              [순차 — 메인]
    ↓
Phase 5: 프리뷰 + 피드백                     [순차 — 메인]
    ↓
Phase 6: 렌더링                             [순차 — 메인]
    ↓
Phase 7: 포스트프로덕션 검증                  [병렬 — post-producer 위임]
    ├→ AI 영상 리뷰 (비주얼/페이싱/싱크/효과)
    ├→ QC 기술 검증 (코덱/LUFS/해상도)
    └→ 접근성 검사 (자막/색상대비/깜빡임)
```

### `/create-short` 파이프라인

`/create-video`와 동일한 병렬 구조, 단 숏폼 특화:
- 9:16 세로 포맷 강제
- 자막 필수 (`/add-captions` 자동 포함)
- Phase 3에서 portrait 방향 에셋만 검색

## 서브에이전트 호출 방법

### media-scout 호출

비주얼 에셋이 필요한 시점에 media-scout 에이전트를 생성하여 병렬로 에셋을 수집합니다:

```
Agent({
  subagent_type: "media-scout",
  prompt: "프롬프트 시트(docs/06-prompt-sheet.md)를 읽고 다음 에셋을 병렬로 수집해줘:
    1. 스톡 영상: [장면별 키워드] — Pexels에서 검색 → public/footage/
    2. AI 이미지: [프롬프트 목록] — config.yaml 프로바이더로 생성 → public/images/
    3. AI 영상 클립: [프롬프트 목록] — config.yaml 프로바이더로 생성 → public/footage/
    수집 완료 후 에셋 목록을 보고해줘."
})
```

### post-producer 호출

렌더링 완료 후 post-producer 에이전트를 생성하여 병렬 검증합니다:

```
Agent({
  subagent_type: "post-producer",
  prompt: "렌더링된 영상(out/video.mp4)을 검증해줘:
    1. AI 리뷰: 비주얼 품질, 페이싱, 오디오-비주얼 싱크, 플랫폼 적합성
    2. QC 검증: ffprobe로 코덱/해상도/LUFS/파일크기 확인
    3. 접근성: 자막 존재, 색상대비 4.5:1, 깜빡임 3회/초 미만
    검증 결과를 종합 리포트로 보고해줘."
})
```

## 순차 vs 병렬 판단 기준

| 조건 | 처리 방식 |
|------|----------|
| 다음 단계의 입력이 현재 출력에 의존 | 순차 |
| 서로 독립적인 에셋 생성/검색 | 병렬 (media-scout) |
| 서로 독립적인 검증/리뷰 | 병렬 (post-producer) |
| 사용자 승인이 필요 | 순차 (메인에서 직접) |
| 코드 작성/수정 | 순차 (메인에서 직접) |

## 에러 핸들링

- 서브에이전트가 하나의 태스크에서 실패해도 나머지는 계속 진행
- 실패한 태스크만 메인 에이전트에 보고 → 대안 제시 또는 재시도
- 예: 스톡 영상 검색 실패 → AI 이미지로 대체 제안
