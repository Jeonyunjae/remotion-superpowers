# Revision Log — 수정 이력 관리

수정 라운드를 추적하고, 피드백 항목별 상태를 관리합니다. 수정 횟수 제한과 버전 관리를 체계적으로 수행합니다.

## 프로세스

### Step 1: 프로젝트 스코프 확인

```bash
cat docs/00-project-scope.md 2>/dev/null
```

수정 횟수 제한을 확인합니다. 없으면 기본 3회로 설정합니다.

### Step 2: 수정 로그 문서 생성/업데이트

**출력 파일**: `docs/08-revision-log.md`

이미 존재하면 새 라운드를 추가합니다. 없으면 새로 생성합니다.

```bash
cat docs/08-revision-log.md 2>/dev/null || echo "NEW"
```

### Step 3: 피드백 수집

> "수정 피드백을 알려주세요. 가능하면 구체적인 타임코드와 함께 알려주시면 정확하게 반영할 수 있습니다.\n\n예시:\n- 0:05 — 나레이션 속도가 너무 빠름\n- 0:12~0:15 — 이미지를 다른 것으로 교체\n- 전체 — BGM 볼륨 좀 낮춰주세요"

### Step 4: 수정 로그 업데이트

```markdown
# Revision Log

## 프로젝트 정보
- 수정 허용 라운드: [n]회
- 현재 라운드: [n]/[max]

---

## Round 1 — [날짜]
**버전**: v1 (러프컷)
**피드백 접수**: [날짜/시간]

| # | 타임코드 | 피드백 내용 | 우선순위 | 상태 | 수정 내용 |
|---|----------|-----------|---------|------|----------|
| 1 | 0:05 | [피드백] | [높음/중간/낮음] | [대기/진행중/완료/보류] | [수정 설명] |
| 2 | 전체 | [피드백] | [높음/중간/낮음] | [대기/진행중/완료/보류] | [수정 설명] |

**수정 완료일**: [날짜]
**수정 결과**: v2 렌더링 완료

---

## Round 2 — [날짜]
[위와 동일 형식]

---

## 버전 이력

| 버전 | 유형 | 날짜 | 파일 | 비고 |
|------|------|------|------|------|
| v1 | 러프컷 | [날짜] | out/video-v1.mp4 | 최초 렌더링 |
| v2 | 수정본 | [날짜] | out/video-v2.mp4 | Round 1 피드백 반영 |
| v3 | 파인컷 | [날짜] | out/video-v3.mp4 | Round 2 피드백 반영 |
```

### Step 5: 수정 횟수 경고

수정 횟수 제한에 근접하면 경고합니다:

> "⚠️ 현재 수정 라운드 [n]/[max]입니다. [남은 횟수]회 남았습니다."

수정 횟수를 초과하면:

> "⚠️ 수정 허용 횟수([max]회)를 초과했습니다. 추가 수정이 필요하면 프로젝트 스코프를 업데이트해주세요."

### Step 6: 수정 적용

각 피드백 항목에 대해 코드를 수정하고, 상태를 "완료"로 업데이트합니다.

수정 완료 후 버전 번호를 올려 재렌더링합니다:
```bash
npx remotion render [CompositionId] out/video-v[n].mp4
```
