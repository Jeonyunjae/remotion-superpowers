# Receive Brief — 클라이언트 브리프 수령 및 분석

사용자로부터 영상 제작 요청(컨셉, 기획안, 구두 설명 등)을 받아 구조화된 클라이언트 브리프 문서를 작성합니다.

## 프로세스

### Step 1: 입력 수령

사용자의 컨셉/요청을 수령합니다. 텍스트, 문서 설명, 구두 요청 등 어떤 형식이든 가능합니다.

### Step 2: 필수 정보 확인

아래 항목 중 사용자가 제공하지 않은 정보는 **반드시 질문**하여 확인합니다. 한꺼번에 질문하지 말고, 자연스러운 대화로 진행합니다.

| # | 항목 | 질문 예시 |
|---|------|----------|
| 1 | 영상 목적 | "이 영상의 목적이 무엇인가요? (교육, 홍보, 설명, 마케팅, 내부 교육 등)" |
| 2 | 타깃 시청자 | "주요 시청자는 누구인가요? (내부 직원, 일반 대중, 특정 업종 종사자 등)" |
| 3 | 톤앤매너 | "어떤 분위기를 원하시나요? (진지한, 밝은, 다큐멘터리, 캐주얼, 유머러스 등)" |
| 4 | 영상 길이 | "원하시는 영상 길이가 있나요? (30초, 1분, 3분, 5분 등)" |
| 5 | 배포 채널 | "어디에 게시할 예정인가요? (유튜브, 사내 인트라넷, SNS, 프레젠테이션 등)" |
| 6 | 예산 범위 | "AI 생성 도구 비용은 어느 수준까지 가능한가요? (무료 AI만 / 저비용 유료 허용 / 프리미엄)" |
| 7 | 일정/납기 | "완성 희망 일정이 있나요?" |
| 8 | 필수 포함 요소 | "반드시 포함해야 할 내용이나 소재가 있나요? (로고, 특정 장면, 슬로건 등)" |
| 9 | 제약 사항 | "금지되거나 피해야 할 요소가 있나요? (경쟁사 언급, 특정 색상, 민감한 주제 등)" |
| 10 | 기존 소스 유무 | "사용 가능한 기존 소스가 있나요? (촬영 영상, 로고 파일, 이미지, 음원 등)" |

### Step 2.5: 모호함 감지 및 구체화

`rules/questioning-protocol.md`의 모호함 감지 규칙을 적용합니다.

사용자의 답변 중 모호한 표현이 있으면 **선택지를 제시하여 구체화**합니다.

**예시:**
> 사용자: "분위기는 좋은 느낌으로 해주세요"
> 에이전트: "좀 더 구체적으로 정해볼게요. 다음 중 어느 쪽이 가까운가요?
> 1. 밝고 활기찬 느낌 (광고, 홍보 영상)
> 2. 차분하고 신뢰 가는 느낌 (기업, 교육 영상)
> 3. 긴장감 있고 몰입되는 느낌 (안전, 다큐멘터리)
> 4. 따뜻하고 감성적인 느낌 (브랜딩, 스토리텔링)"

> 사용자: "길이는 적당하게"
> 에이전트: "영상 목적과 채널을 고려하면 이 정도가 적당할 것 같은데요:
> 1. 30초~1분 (SNS 숏폼, 빠른 전달)
> 2. 1~3분 (유튜브 요약, 사내 공지)
> 3. 3~5분 (상세 교육, 프레젠테이션)
> 4. 5분 이상 (전체 과정 교육, 다큐멘터리)
> 어느 범위가 맞나요?"

답변이 충분히 구체적일 때까지 질문을 반복합니다. 모든 항목이 명확해진 후에만 Step 3으로 진행합니다.

### Step 3: 브리프 문서 작성

모든 정보가 수집되면 아래 형식으로 `docs/01-client-brief.md`를 생성합니다.

```bash
mkdir -p docs
```

**출력 파일**: `docs/01-client-brief.md`

```markdown
# Client Brief

## 프로젝트명
[프로젝트명 — 사용자 요청에서 도출하거나 함께 결정]

## 목적
[영상의 목적 — 교육, 홍보, 설명, 마케팅, 내부 교육 등]

## 타깃 시청자
[주요 시청자 정의 — 인구통계, 직군, 관심사 등]

## 톤앤매너
[원하는 분위기 — 진지한, 밝은, 다큐멘터리, 캐주얼 등]

## 영상 길이
[목표 러닝타임]

## 배포 채널
[유튜브, 사내, SNS, 프레젠테이션 등 — 복수 가능]

## 예산
[무료 AI만 / 저비용 유료 허용 / 프리미엄 — config.yaml 프리셋과 연동]

## 일정
[완성 희망일 또는 마감 일정]

## 필수 포함 요소
- [항목 1]
- [항목 2]
- ...

## 제약 사항
- [금지/회피 요소 1]
- [금지/회피 요소 2]
- ...

## 기존 소스
- [사용 가능한 기존 자료 목록]
- [파일 경로 또는 URL]

## 특이사항
[기타 참고사항, 레퍼런스 영상 URL, 벤치마크 등]
```

### Step 4: 사용자 확인

작성된 브리프를 사용자에게 보여주고 확인을 받습니다:

> "클라이언트 브리프를 작성했습니다. 내용을 확인해주세요. 수정할 부분이 있으면 말씀해주세요. 확인되면 다음 단계(`/creative-brief`)로 진행합니다."

## 다음 단계

브리프 확인이 완료되면 `/creative-brief`를 실행하여 내부 크리에이티브 브리프를 작성합니다.
