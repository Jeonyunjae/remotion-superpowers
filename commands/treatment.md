# Treatment — 트리트먼트 (비주얼 방향 제안)

클라이언트 브리프와 크리에이티브 브리프를 바탕으로 영상의 비주얼 방향, 스타일, 무드, 전체 크리에이티브 비전을 제안합니다.

**이 단계는 승인 체크포인트입니다.** 사용자의 승인 없이 다음 단계로 진행할 수 없습니다.

## 사전 조건

다음 파일이 존재해야 합니다:
- `docs/01-client-brief.md`
- `docs/02-creative-brief.md`

없으면 해당 커맨드를 먼저 실행하도록 안내합니다.

## 프로세스

### Step 1: 이전 문서 읽기

```bash
cat docs/01-client-brief.md
cat docs/02-creative-brief.md
```

### Step 2: 비주얼 방향 질문

비주얼 방향을 결정하기 위해 사용자에게 **질문**합니다. `rules/questioning-protocol.md`의 모호함 감지 규칙을 적용합니다.

**필수 질문 항목:**

| # | 항목 | 질문 |
|---|------|------|
| 1 | 영상 형식 | "영상의 형식은 어떤 것을 원하시나요?\n1. 실사 촬영 느낌 (AI 생성 실사풍)\n2. 2D 일러스트/애니메이션\n3. 3D 렌더링/모션그래픽\n4. 혼합 (실사 + 그래픽 오버레이)" |
| 2 | 색감 | "전체적인 색감은 어떤 방향이 좋을까요?\n1. 밝고 선명한 색감 (비비드)\n2. 부드럽고 따뜻한 색감 (웜톤)\n3. 차갑고 전문적인 색감 (쿨톤)\n4. 어둡고 시네마틱한 색감 (다크)" |
| 3 | 레퍼런스 | "참고하고 싶은 영상이나 이미지가 있나요? (URL이나 설명으로 알려주세요)" |
| 4 | 인물 등장 | "영상에 사람/캐릭터가 등장하나요? 등장한다면 어떤 인물인가요?" |
| 5 | 나레이션 목소리 | "나레이션 목소리는 어떤 느낌이 좋을까요?\n1. 남성 — 차분하고 권위 있는\n2. 남성 — 친근하고 편안한\n3. 여성 — 밝고 전문적인\n4. 여성 — 따뜻하고 부드러운" |
| 6 | 음악 | "배경 음악은 어떤 장르/분위기가 좋을까요?\n1. 잔잔한 피아노/어쿠스틱\n2. 시네마틱 오케스트라\n3. 모던 일렉트로닉\n4. 밝은 팝/경쾌한 느낌" |

**조건부 질문:**
- 인물이 등장하는 경우: "등장인물의 연령대, 복장, 표정 방향을 알려주세요"
- 특정 장소가 배경인 경우: "배경이 되는 장소의 분위기를 구체적으로 설명해주세요 (시간대, 조명, 날씨 등)"
- 텍스트 오버레이가 필요한 경우: "텍스트 스타일은 어떤 것이 좋을까요? (폰트 느낌, 색상, 크기 방향)"

모든 비주얼 결정이 확정된 후 트리트먼트를 작성합니다.

### Step 3: 트리트먼트 작성

**출력 파일**: `docs/03-treatment.md`

```markdown
# Treatment

## 1. 전체 내러티브 구조

### 도입부 (Opening)
[영상의 첫 인상 — 어떻게 시작하는지]

### 전개 (Development)
[핵심 내용을 어떤 순서로 전달하는지]

### 클라이맥스 (Climax)
[가장 임팩트 있는 순간]

### 마무리 (Closing)
[어떻게 끝맺는지, CTA 포함]

## 2. 비주얼 스타일

### 영상 형식
[실사 / 2D 일러스트 / 3D / 모션그래픽 / 혼합 — 구체적 비율과 이유]

### 화면 비율
[16:9 / 9:16 / 1:1 — 배포 채널에 따라]

### 해상도/FPS
[1920x1080 / 1080x1920 등, 30fps / 60fps]

## 3. 색감 및 조명 방향

### 색상 팔레트
- 주요 색상: [hex 코드 + 설명]
- 보조 색상: [hex 코드 + 설명]
- 배경 톤: [밝은/어두운/그라데이션]

### 조명 방향
[자연광/스튜디오/시네마틱/네온 등]

## 4. 무드 레퍼런스
[참고할 수 있는 영상, 이미지, 영화 장면 등 — URL이나 설명]
- 레퍼런스 1: [설명]
- 레퍼런스 2: [설명]

## 5. 음악 및 사운드 방향

### 배경음악
- 장르: [로파이, 시네마틱, 일렉트로닉, 오케스트라 등]
- 무드: [차분한, 에너지 넘치는, 긴장감 있는 등]
- 템포: [BPM 범위 또는 느린/보통/빠른]
- 보컬: [없음 — AI 음악 생성 시 항상 "no vocals"]

### 효과음
- 필요한 효과음 목록: [목록]
- 효과음 스타일: [리얼리스틱/스타일라이즈드]

### 나레이션
- 화자 성격: [전문적/친근한/권위있는]
- 음성 특성: [남/여, 높낮이, 속도]
- 언어: [한국어/영어/기타]

## 6. 캐릭터 및 인물 방향
[영상에 등장하는 인물/캐릭터가 있는 경우]
- 등장인물 수: [명]
- 각 인물의 역할: [설명]
- 복장/스타일: [설명]
- AI 생성 시 주의사항: [일관성 유지 방법 등]

## 7. 편집 스타일
- 컷 페이스: [빠른/보통/느린]
- 전환 효과: [하드컷/디졸브/슬라이드/특수효과]
- 텍스트 오버레이: [사용 여부, 스타일]
- 타이포그래피: [폰트 방향, 크기, 배치]

## 8. AI 도구 활용 계획
- 이미지 생성: [어떤 장면에 AI 이미지 사용]
- 영상 클립 생성: [어떤 장면에 AI 영상 사용]
- 스톡 영상: [어떤 장면에 스톡 사용]
- 예상 비용: [config.yaml 프리셋 기준]
```

### Step 4: 승인 요청

트리트먼트를 사용자에게 제시하고 **반드시 승인을 받습니다**:

> "트리트먼트를 작성했습니다. 이 방향으로 진행할까요? 수정이 필요한 부분이 있으면 말씀해주세요."

**승인 시**: 다음 단계(`/write-script`)로 진행
**수정 요청 시**: 피드백 반영 후 다시 승인 요청
**거절 시**: 대안 방향을 제안하거나 요구사항 재확인
