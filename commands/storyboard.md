# Storyboard — 스토리보드 작성

스크립트의 각 장면/샷에 대해 사용자에게 피사체, 인물, 배경, 동선, 카메라를 **질문하여 확인**한 후, 상세 스토리보드를 작성하고 AI 프리뷰 이미지를 생성합니다.

**이것이 프리프로덕션에서 가장 중요한 단계입니다.** 이전 방식은 비주얼 디테일을 확인하지 않고 바로 생성에 들어가서 결과물 품질이 낮았습니다. 이 단계에서 모든 시각적 세부사항을 확정합니다.

**이 단계는 승인 체크포인트입니다.** 사용자의 승인 없이 다음 단계로 진행할 수 없습니다.

## 사전 조건

다음 파일이 모두 존재해야 합니다:
- `docs/01-client-brief.md`
- `docs/02-creative-brief.md`
- `docs/03-treatment.md`
- `docs/04-script.md`

없으면 해당 커맨드를 먼저 실행하도록 안내합니다.

## 프로세스

### Step 1: 스크립트 읽기

```bash
cat docs/04-script.md
cat docs/03-treatment.md
```

### Step 2: 장면별 질문 (핵심 단계)

스크립트의 **각 씬/샷에 대해 사용자에게 다음을 질문**합니다. 절대 추측하지 않습니다.

한 장면씩 순서대로 진행합니다. 한꺼번에 모든 장면을 질문하지 않습니다.

**각 장면에 대한 질문 목록:**

> **Scene [번호]: [씬 제목]**
> 스크립트 내용: "[해당 씬의 화면 설명 요약]"
>
> 1. **피사체**: 이 장면에 등장하는 피사체는 무엇인가요? (사람, 캐릭터, 물체, 환경, 텍스트 등)
> 2. **인물** (인물이 있는 경우):
>    - 몇 명이 등장하나요?
>    - 어떤 복장인가요?
>    - 어떤 표정인가요?
>    - 어떤 동작/포즈인가요?
> 3. **동선**: 인물이나 피사체의 움직임은? (어디서 어디로 이동? 정지?)
> 4. **카메라 앵글**: 어떤 앵글로 촬영하나요? (정면, 위에서 내려다보기, 아래에서 올려다보기, 클로즈업, 와이드샷, 미디엄샷)
> 5. **카메라 움직임**: 카메라가 움직이나요? (고정, 패닝, 틸트, 줌인/아웃, 트래킹)
> 6. **배경 환경**: 배경은 어떤 모습인가요? (실내/실외, 구체적 장소, 날씨, 조명, 시간대)
> 7. **화면 전환**: 이전 장면에서 어떻게 전환되나요? (하드컷, 디졸브, 페이드, 슬라이드, 와이프)
> 8. **특수 효과**: 추가로 필요한 시각 효과가 있나요? (파티클, 텍스트 애니메이션, 오버레이 등)

사용자의 답변을 받은 후 다음 장면으로 넘어갑니다.

**사용자가 "알아서 해줘" 또는 모호하게 답변하는 경우:**

`rules/questioning-protocol.md`의 모호함 감지 규칙을 적용합니다.

1. 모호 신호를 감지하면 **구체적인 선택지 2~3개**를 먼저 제시합니다
2. 각 선택지의 차이점과 영상에서의 효과를 설명합니다
3. 사용자가 하나를 선택하면 그 방향으로 확정합니다
4. 절대 선택 없이 진행하지 않습니다

> 사용자: "카메라는 알아서 해줘"
> 에이전트: "이 장면의 내용을 고려하면 이런 선택지가 있어요:
> 1. 와이드샷으로 전체 환경을 보여준 후 → 인물로 줌인 (공간감 + 집중)
> 2. 미디엄샷 고정으로 안정적으로 (교육/설명에 적합)  
> 3. 인물 클로즈업에서 → 천천히 줌아웃하며 환경 공개 (드라마틱)
> 어느 쪽이 좋을까요?"

### Step 3: 스토리보드 문서 작성

모든 장면의 정보가 수집되면 스토리보드를 작성합니다.

```bash
mkdir -p docs/storyboard-previews
```

**출력 파일**: `docs/05-storyboard.md`

```markdown
# Storyboard

## 기본 정보
- 총 샷 수: [개수]
- 총 러닝타임: [시간]
- 화면 비율: [비율]

---

### Shot 1
- **시간**: 0:00 ~ 0:05
- **씬**: Scene 1 - [씬 제목]
- **피사체**: [사용자가 답변한 피사체 — 구체적으로]
- **인물**: [등장인물 — 복장, 표정, 동작 상세]
- **동선**: [움직임 설명 — 시작 위치 → 끝 위치]
- **카메라**: [앵글 + 움직임 — 예: "미디엄샷, 천천히 줌인"]
- **배경**: [배경 환경 상세 — 장소, 조명, 분위기]
- **전환**: [이전 장면에서의 전환 방식]
- **오디오**: [이 시점의 나레이션/음악/효과음]
- **화면 텍스트**: [오버레이 텍스트가 있으면 기재]
- **특수 효과**: [있으면 기재]
- **프리뷰 이미지**: `storyboard-previews/shot-01.png`

---

### Shot 2
[위와 동일한 형식으로 반복]

---

[모든 샷에 대해 반복]

---

## 전체 흐름 요약

| Shot | 시간 | 주요 내용 | 전환 |
|------|------|----------|------|
| 1 | 0:00~0:05 | [요약] | [전환] |
| 2 | 0:05~0:10 | [요약] | [전환] |
| ... | ... | ... | ... |
```

### Step 4: AI 프리뷰 이미지 생성

각 샷에 대해 AI 이미지를 생성하여 시각적 레퍼런스를 제공합니다.

`config.yaml`에서 설정된 이미지 프로바이더를 사용합니다.

각 샷의 스토리보드 정보(피사체, 인물, 배경, 카메라 앵글)를 조합하여 이미지 프롬프트를 생성합니다.

생성된 이미지를 `docs/storyboard-previews/shot-XX.png` 형식으로 저장합니다.

### Step 5: 승인 요청

완성된 스토리보드와 프리뷰 이미지를 사용자에게 제시하고 **반드시 승인을 받습니다**:

> "스토리보드를 완성했습니다. 각 장면의 구성과 프리뷰 이미지를 확인해주세요. 수정이 필요한 샷이 있으면 번호로 말씀해주세요."

**승인 시**: 다음 단계(`/prompt-sheet`)로 진행
**수정 요청 시**: 해당 샷만 수정하고 다시 승인 요청
