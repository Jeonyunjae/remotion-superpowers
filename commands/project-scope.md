# Project Scope — 프로젝트 범위 정의

프리프로덕션 시작 전에 프로젝트의 범위, 딜리버러블, 수정 횟수, 일정, 예산을 확정합니다.

## 프로세스

### Step 1: 프로젝트 개요 질문

`rules/questioning-protocol.md`의 모호함 감지 규칙을 적용합니다.

**필수 질문 항목:**

| # | 항목 | 질문 |
|---|------|------|
| 1 | 프로젝트 유형 | "어떤 유형의 영상인가요?\n1. 기업 홍보/마케팅\n2. 교육/연수\n3. 제품/서비스 소개\n4. 유튜브/SNS 콘텐츠\n5. 내부 커뮤니케이션\n6. 기타" |
| 2 | 딜리버러블 | "최종 납품물로 필요한 것을 모두 알려주세요:\n- 영상 수: 몇 개?\n- 포맷: 가로(16:9)? 세로(9:16)? 정방형(1:1)? 복수?\n- 길이: 각각 몇 초/분?" |
| 3 | 수정 횟수 | "수정은 몇 라운드까지 가능한가요? (업계 표준: 2~3회)\n1. 2회 (표준)\n2. 3회 (여유 있음)\n3. 무제한 (개인 프로젝트)" |
| 4 | 일정 | "마감 일정이 있나요? 중간 마일스톤도 있으면 알려주세요." |
| 5 | 예산 | "AI 도구 사용 예산은 어느 수준인가요?\n1. 무료 AI만 사용 ($0)\n2. 저비용 유료 허용 (~$1 이내)\n3. 품질 우선 ($5 이내)\n4. 제한 없음" |
| 6 | 소유권 | "완성된 영상의 소유권과 사용 범위는?\n1. 내부 사용만\n2. 외부 공개 (SNS/유튜브)\n3. 상업적 활용 (광고, 판매)\n4. 전체 권리 양도" |
| 7 | 이해관계자 | "이 영상의 승인 권한자는 누구인가요? (본인만? 상사? 클라이언트?)" |

### Step 2: 마일스톤 일정 수립

프로젝트 유형과 마감일을 기반으로 마일스톤 일정을 제안합니다.

**표준 마일스톤:**
- M1: 브리프 확정
- M2: 크리에이티브 방향 승인 (Treatment)
- M3: 스크립트 승인
- M4: 스토리보드 승인
- M5: 러프컷 리뷰
- M6: 파인컷/최종 승인
- M7: 납품

### Step 3: 스코프 문서 작성

```bash
mkdir -p docs
```

**출력 파일**: `docs/00-project-scope.md`

```markdown
# Project Scope

## 프로젝트 개요
- 프로젝트명: [명칭]
- 유형: [유형]
- 클라이언트/요청자: [이름/조직]

## 딜리버러블
| # | 항목 | 포맷 | 길이 | 비고 |
|---|------|------|------|------|
| 1 | [영상명] | [16:9/9:16/1:1] | [시간] | [비고] |

## 수정 정책
- 허용 라운드: [n]회
- 각 라운드 최대 피드백 항목: [n]개 권장
- 추가 수정 시: [조건]

## 일정
| 마일스톤 | 예상 일자 | 상태 |
|----------|----------|------|
| M1: 브리프 확정 | [날짜] | - |
| M2: 크리에이티브 방향 승인 | [날짜] | - |
| M3: 스크립트 승인 | [날짜] | - |
| M4: 스토리보드 승인 | [날짜] | - |
| M5: 러프컷 리뷰 | [날짜] | - |
| M6: 최종 승인 | [날짜] | - |
| M7: 납품 | [날짜] | - |

## 예산
- AI 도구 프리셋: [free/budget/premium]
- 예상 비용: [금액]
- 예산 상한: [금액]

## 소유권 및 사용 범위
- 소유권: [설명]
- 사용 범위: [설명]
- 크레딧 표시: [필요/불필요]

## 승인 권한자
- [이름/역할]
```
