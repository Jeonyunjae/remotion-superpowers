# Fact Check — 데이터 정확성 검증

스크립트에 포함된 수치, 통계, 인용, 사실 주장의 정확성을 검증합니다. 교육 영상, 인포그래픽, 기업 소개 등 사실 기반 콘텐츠에서 특히 중요합니다.

## 사전 조건

`docs/04-script.md`가 존재해야 합니다. 없으면:
> "스크립트가 없습니다. 먼저 `/write-script`를 실행해주세요."

## 프로세스

### Step 1: 스크립트 읽기

```bash
cat docs/04-script.md
```

### Step 2: 팩트 추출

스크립트에서 검증이 필요한 항목을 추출합니다:
- 수치/통계 (예: "사고 발생률이 30% 감소")
- 날짜/연도 (예: "2024년부터 시행")
- 인용/발언 (예: "WHO에 따르면...")
- 기술적 사실 (예: "나셀은 지상 80m 이상에 위치")
- 법률/규정 (예: "산업안전보건법 제XX조")

### Step 3: 사용자에게 출처 확인

`rules/questioning-protocol.md`의 모호함 감지 규칙을 적용합니다.

> "스크립트에서 검증이 필요한 사실 [n]개를 발견했습니다. 각 항목의 출처를 확인하겠습니다."

각 항목에 대해:
> "[사실 내용] — 이 정보의 출처가 있나요? (문서, URL, 내부 데이터 등)"

### Step 4: 팩트체크 문서 작성

```bash
mkdir -p docs
```

**출력 파일**: `docs/04-fact-check.md`

```markdown
# Fact Check Report

## 검증 요약
- 총 검증 항목: [n]개
- 확인 완료: [n]개
- 출처 불명: [n]개
- 수정 필요: [n]개

## 검증 상세

| # | 스크립트 내용 | 출처 | 검증 결과 | 비고 |
|---|-------------|------|----------|------|
| 1 | [사실 내용] | [출처] | 확인됨/출처 불명/수정 필요 | [비고] |
| 2 | ... | ... | ... | ... |

## 수정 권고사항
- [수정이 필요한 항목과 올바른 정보]

## 면책 조항
- [출처가 불확실한 항목에 대한 주의 사항]
```

### Step 5: 스크립트 수정 제안

수정이 필요한 항목이 있으면 스크립트 수정을 제안합니다.

> "팩트체크 결과 [n]개 항목의 수정이 권고됩니다. 스크립트를 수정할까요?"
